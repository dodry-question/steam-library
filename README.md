# Steam Smart Library

Веб-приложение для анализа библиотеки Steam. Проект позволяет просматривать свою коллекцию игр, актуальные цены в магазине, время в игре, а также получать персонализированные рекомендации с помощью большой языковой модели (LLM).

Встроенная нейросеть анализирует ваш профиль и предлагает игры. 
Есть два режима работы:
*   **Глобальный:** ИИ смотрит на топ-15 ваших любимых игр и подбирает новые, опираясь на выбранное настроение (например, "Скрытые жемчужины" или "Медитативное").
*   **Локальный (Поиск похожего):** Вы можете выделить конкретные игры в библиотеке (Shift + Клик на ПК или долгое нажатие на телефоне), и ИИ найдет игры, похожие именно на этот набор.

## Как это работает под капотом

Проект построен на связке **Python (FastAPI)** и чистого **JavaScript**.

### Фоновая загрузка и кэширование (База данных)
Чтобы не превышать лимиты (Rate Limits) официального API Steam и ускорить загрузку, реализована система локального кэширования:
1. При запросе библиотеки бэкенд быстро отдает список ID игр пользователя.
2. Фронтенд рисует "скелеты" карточек.
3. Бэкенд через асинхронный генератор (StreamingResponse) проверяет SQLite базу (`games.db`). Если данные свежие (сохранены после последнего обновления магазина Steam в 10:10 PT), они отдаются из БД.
4. Если игры в базе нет, сервер делает запрос к Steam Store API, парсит цену, скидки и жанры, сохраняет в базу и отправляет на фронтенд. 

### Интеграция с LLM (Генерация рекомендаций)
За рекомендации отвечает модель `gpt-4o-mini` (через API-провайдера). 
Как строится промпт:
*   Система собирает названия любимых игр пользователя.
*   **Защита от дублей:** В промпт передается список из топ-300 игр пользователя с жестким запретом (system prompt) советовать то, что уже куплено.
*   **Защита от повторений:** Сохраняется история выданных рекомендаций в рамках сессии, чтобы при повторном нажатии ИИ предлагал новые варианты.
*   **Парсинг:** Модель проинструктирована возвращать ответ строго в формате JSON. Бэкенд валидирует этот JSON и ищет предложенные названия в базе Steam, чтобы вернуть на фронтенд красивые кликабельные карточки с реальными обложками.

### Система авторизации
Доступно два метода:
1.  **OpenID (Официальный вход):** Пользователь авторизуется на стороне Steam, сервер получает только публичный `SteamID64`.
2.  **По ссылке (Vanity URL):** Пользователь может ввести кастомную ссылку на профиль (например, `steamcommunity.com/id/nickname`). Бэкенд использует метод `ResolveVanityURL` для получения `SteamID64`.

## Локальный запуск (Для разработчиков)

### 1. Подготовка окружения

Клонируйте репозиторий и перейдите в папку проекта:
```bash
git clone https://github.com/dodry-question/steam-library.git
cd steam-library
```

Создайте и активируйте виртуальное окружение:

* **Для Windows (CMD/PowerShell):**
  ```bash
  python -m venv venv
  venv\Scripts\activate
  ```
* **Для macOS/Linux (GitBash):**
  ```bash
  python -m venv venv
  source venv/bin/activate
  ```

Установите необходимые зависимости:
```bash
pip install -r requirements.txt
```

### 2. Настройка переменных окружения

Перед запуском серверу нужны API ключи. Создайте файл `.env` в корневой папке проекта (рядом с `main.py`) и скопируйте туда следующие параметры, подставив свои значения:

```env
# --- Основные настройки ---
STEAM_API_KEY=ваш_ключ_от_steam
MY_DOMAIN=http://localhost:8000

# --- Основная LLM (VseGPT) ---
VSEGPT_API_KEY=ваш_ключ_от_vsegpt
VSEGPT_BASE_URL=https://api.vsegpt.ru/v1

# --- Альтернативные провайдеры LLM (Опционально) ---
GROQ_API_KEY=ваш_ключ_от_groq
OPENROUTER_API_KEY=ваш_ключ_от_openrouter
```
*(Ключ Steam можно получить на [портале разработчиков Steam](https://steamcommunity.com/dev/apikey))*

### 3. Запуск сервера

Когда зависимости установлены и ключи добавлены, запустите приложение:
```bash
python main.py
```

Сайт будет доступен в браузере по адресу: **http://localhost:8000**
